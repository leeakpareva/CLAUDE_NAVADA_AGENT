10.5 How Jenkins Works in the NAVADA Network

Jenkins provides continuous integration and deployment across the NAVADA mesh:

  1. Trigger: Git push to GitHub, scheduled cron, or manual trigger from Jenkins UI
  2. Build: Jenkins controller (HP) dispatches job to local or remote agent (EC2)
  3. Test: Agent runs unit tests, linting, type checks in isolated environment
  4. Deploy: On success, artifacts are pushed to target node and PM2 service restarted
  5. Notify: Result sent to Telegram via bot webhook for real-time alerts

Network integration:

  • Jenkins controller on HP communicates with EC2 agent via Tailscale (100.98.118.33)
  • Nginx reverse proxy on HP provides /jenkins/ path routing from LAN and Cloudflare
  • Jenkins shares the Docker network (infrastructure_navada-net) with Nginx for internal routing
  • Build artifacts can be deployed to any node in the Tailscale mesh via SSH/SCP

10.6 Setting Up a Pipeline

1. Open Jenkins UI: http://192.168.0.58:8080/jenkins/
2. Click "New Item" > Enter name > Select "Pipeline" > OK
3. Under "Pipeline", select "Pipeline script from SCM"
4. SCM: Git
5. Repository URL: https://github.com/Navada25/your-repo.git
6. Branch: */main
7. Script Path: Jenkinsfile
8. Save

Example Jenkinsfile:
--------------------
pipeline {
    agent any
    stages {
        stage('Install') {
            steps { sh 'npm ci' }
        }
        stage('Test') {
            steps { sh 'npm test' }
        }
        stage('Deploy') {
            steps { sh 'pm2 restart service-name' }
        }
    }
    post {
        always {
            sh 'curl -s "https://api.navada-edge-server.uk/telegram/"'
        }
    }
}

11. FAILOVER SYSTEM (HP TO ORACLE)

NORMAL OPERATION: Internet --> Cloudflare Tunnel (HP 7c9e3c36...) --> HP Nginx --> PM2 services HP failover-sync -----(SCP every 5 min)-----> Oracle VM EC2 health-monitor ---(ping every 5 min)----> HP HP GOES DOWN (detected after 3 failures = 15 min): EC2 ---SSH---> Oracle VM ---runs---> failover-activate.sh 1. Cloudflare DNS switches api.navada-edge-server.uk FROM: HP tunnel (7c9e3c36...) TO: Oracle tunnel (96e30aeb...) 2. Oracle starts: Docker (Nginx + Cloudflared), PM2 (bot, inbox, trading) 3. Telegram notification: "FAILOVER ACTIVE" HP RECOVERS: EC2 detects HP is back --> Oracle failover-deactivate.sh 1. State synced back to HP 2. Oracle services stopped 3. DNS restored to HP tunnel 4. Telegram notification: "FAILBACK COMPLETE"
