1. Executive Summary

NAVADA Edge is an AI-as-a-Service platform that delivers bespoke, autonomous AI agents to small businesses, solo consultants, and agencies. Each client receives a dedicated specialist AI agent exclusively powered by Anthropic Claude, running 24/7 on Telegram, with skills and behaviour custom-built to their brand and use case. NAVADA is an Anthropic-only platform — Claude is the sole AI model used across all Edge and IoT deployments.

Key Value Proposition

NAVADA Edge fills a gap in the market between generic AI chatbots (ChatGPT, Claude Pro) and enterprise-grade AI platforms (Sierra, 11x AI). For £200/month, clients get what would cost £50K-100K to build with a development team, or £5K-15K/month from an enterprise vendor.

2. Product Definition

What Is NAVADA Edge?

NAVADA Edge is an AI home server deployment service. Clients receive a fully autonomous AI agent that operates as their personal or business assistant, accessible via Telegram 24/7. Each agent is:

  • Exclusively powered by Anthropic Claude — frontier-level AI reasoning, analysis, and generation. NAVADA is an Anthropic-only platform.
  • Custom-built — specialist skills, behaviour, personality, and knowledge tailored to the client's brand and workflow
  • Autonomous — runs independently, responds in real-time, executes tasks, manages information
  • Always-on — 24/7 availability via Telegram on any device (phone, tablet, desktop)

Example Agent Types

Agent TypeTarget ClientKey Skills
Research AssistantConsultants, analystsWeb research, report generation, data analysis, briefings
Sales AssistantAgencies, freelancersLead research, email drafting, CRM updates, proposal generation
Operations AssistantSmall business ownersScheduling, task management, reporting, process automation
Content AssistantMarketing agencies, creatorsCopy generation, social media drafting, image generation, SEO
Property AssistantEstate agents, landlordsMarket reports, tenant comms, listing descriptions, compliance checks
Recruitment AssistantRecruiters, HR firmsCandidate sourcing, JD writing, market mapping, outreach

What Every Agent Includes

  • Dedicated Telegram bot (client's own bot, branded)
  • Custom system prompt defining personality, skills, and boundaries
  • Persistent conversation memory
  • Tool use: web search, file generation, image creation (DALL-E), email drafts
  • Knowledge base: client-specific documents, FAQs, brand guidelines loaded
  • Model switching: Sonnet (fast daily tasks) / Opus (complex analysis)
  • Comprehensive e2e documentation and how-to guide

Client Deliverables

  1. Live AI agent on Telegram (branded, custom skills)
  2. Onboarding session (30-60 min walkthrough)
  3. End-to-end documentation pack (PDF)
  4. How-to guide for daily use
  5. Knowledge base pre-loaded with client materials

Day 1: Discovery & Payment

  1. Discovery call (30 min): Understand client's business, use case, desired agent persona, key skills needed
  2. Brief document: Capture agent name, personality, skills list, knowledge base sources, brand guidelines
  3. Payment: £2,000 setup fee via Stripe Checkout (or Stripe Invoice for business clients)
  4. Subscription activation: £200/mo recurring starts immediately via Stripe Subscriptions

Day 1-2: Agent Build

  1. Create new Telegram bot via BotFather (client's branded bot name)
  2. Configure system prompt (persona, skills, boundaries, brand voice)
  3. Set up knowledge base folder with client materials
  4. Configure tools (web search, file generation, email, image generation as needed)
  5. Register slash commands with Telegram autocomplete
  6. Set up PM2 process on NAVADA Edge server
  7. Configure conversation memory and interaction logging
  8. Configure per-client cost tracking
  9. Testing: functional tests, edge cases, brand compliance

Day 2: Handover

  1. Onboarding session (30-60 min): Live walkthrough of the agent, demo all commands, answer questions
  2. Deliver documentation pack: e2e guide, command reference, troubleshooting tips
  3. Client confirms: Agent is live and working as expected

Process Flow (Tier 2: Self-Hosted)

Same as above, plus:

  1. Client purchases hardware (laptop, ethernet cable) — NAVADA provides spec recommendations
  2. Remote setup session via TeamViewer: OS config, Node.js, PM2, Tailscale, Anthropic API key
  3. Deploy agent to client's hardware
  4. Configure Tailscale for remote access
  5. Verify 24/7 operation, test fail-over scenarios
  6. Deliver extended documentation: hardware management guide, restart procedures, API key rotation

Disconnection Policy (Hosted Clients)

Payment Grace Period: Clients have 7 days from a missed payment to resolve the issue. After 7 days, the agent is automatically disconnected via PM2 stop. The agent can be reactivated immediately upon payment of outstanding balance. No data is deleted for 30 days after disconnection.

Client Agent Provisioning (Repeatable Process)

  1. Create Telegram bot via BotFather → obtain bot token
  2. Copy client-bot-template.js → client-bots/clientname-bot.js
  3. Configure: bot token, system prompt, tool permissions, KB path
  4. Create KB directory: client-bots/kb/clientname/
  5. Load client documents into KB
  6. Add to ecosystem.config.js: new PM2 app entry
  7. Start: pm2 start ecosystem.config.js --only clientname-bot
  8. Test: send messages, verify all tools, check cost logging
  9. Register slash commands with Telegram
  10. Create Stripe subscription for client
  11. Send documentation pack to client
  12. Onboarding call

Monthly Processes

  • Review all client usage metrics and API costs
  • Generate per-client cost vs revenue report
  • Send monthly usage summary to clients (optional, builds trust)
  • Check for KB updates needed (client feedback)
  • Review and update agent skills based on client usage patterns
  • Stripe revenue reconciliation
  • Proactive client check-in for retention
