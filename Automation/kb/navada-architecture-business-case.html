5. Disaster Recovery (Implemented Today)

ScenarioDetectionSwitchoverTotal Downtime
HP crash/power outage15 min (3 health checks)~2 min~17 min
Manual /failover commandInstant~2 min~2 min
Planned maintenancePre-triggered~2 min~2 min

What fails over automatically:

  • Telegram Bot (Claude Chief of Staff, primary control interface)
  • Inbox Auto-Responder (external email replies)
  • Trading API + Scheduler (market operations)

How it works:

  1. HP syncs state to Oracle every 5 minutes (SCP over Tailscale)
  2. EC2 watchdog detects HP offline (3 failed pings)
  3. EC2 SSHs into Oracle, runs failover-activate.sh
  4. Cloudflare DNS switches api.navada-edge-server.uk to Oracle tunnel
  5. Oracle starts dormant services via PM2 + Docker
  6. Lee receives Telegram notification: "FAILOVER ACTIVE"

When HP recovers, the process reverses automatically: state syncs back, services stop on Oracle, DNS restores to HP.
